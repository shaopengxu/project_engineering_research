# Todo CLI 产品需求文档

## 1. 项目概述

### 1.1 背景
开发者在日常工作中需要快速记录和管理待办事项。现有的 Todo 工具多为 GUI 应用或 Web 服务，需要离开终端操作。本项目提供一个轻量的命令行 Todo 工具，让开发者在终端内即可完成待办管理。

### 1.2 目标用户
日常使用终端工作的开发者。

### 1.3 核心价值
在终端内快速管理待办事项，无需切换上下文。

## 2. 功能需求

### 2.1 任务管理

**描述**: 提供待办事项的增删改查和状态管理功能。

**功能点**:
- [ ] 添加任务：通过命令行快速添加一条待办
- [ ] 列出任务：查看所有待办，支持按状态筛选
- [ ] 标记完成：将指定任务标记为已完成
- [ ] 删除任务：删除指定任务

**业务规则**:
- 任务 ID 使用自增整数，从 1 开始
- 新任务默认状态为 todo
- 任务状态只有两种：todo 和 done
- 标记完成是单向操作，不支持恢复为 todo
- 删除任务后 ID 不复用

**验收标准**:
- [ ] 用户能通过 `todo add "买牛奶"` 添加一条待办
- [ ] 用户能通过 `todo list` 查看所有待办，显示 ID、状态、标题
- [ ] 用户能通过 `todo list --status done` 只查看已完成的待办
- [ ] 用户能通过 `todo done 1` 将 ID 为 1 的任务标记为完成
- [ ] 用户能通过 `todo delete 1` 删除 ID 为 1 的任务
- [ ] 操作不存在的任务 ID 时给出明确错误提示

### 2.2 数据持久化

**描述**: 将任务数据持久化到本地 JSON 文件，支持自定义存储路径。

**功能点**:
- [ ] JSON 文件存储：任务数据以 JSON 格式存储在本地文件中
- [ ] 存储路径可配置：通过环境变量 `TODO_FILE` 自定义存储路径

**业务规则**:
- 默认存储路径为 `~/.todo.json`
- 如果 `TODO_FILE` 环境变量存在且非空，使用该路径
- 首次使用时自动创建存储文件
- 每次写操作后立即持久化，不缓存

**验收标准**:
- [ ] 首次运行时自动创建 `~/.todo.json`
- [ ] 关闭终端后重新打开，之前添加的任务仍然存在
- [ ] 设置 `TODO_FILE=/tmp/my-todo.json` 后，数据存储到指定路径

## 3. 非功能需求

- 性能: 启动时间 < 200ms（空数据和 100 条数据下均满足）
- 兼容性: 支持 macOS 和 Linux

## 4. 不在范围内

- 云同步
- 多用户协作
- GUI 界面
- 任务分类 / 标签
- 截止日期 / 提醒

## 5. 优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 添加任务、列出任务 | 最基本的 CRUD，缺一不可 |
| P0 | JSON 文件持久化 | 没有持久化工具无法使用 |
| P1 | 标记完成、删除任务 | 核心交互，紧随 P0 实现 |
| P1 | 按状态筛选 | 任务多了之后的基本需求 |
| P2 | JSON 输出格式 | 便于与其他工具组合使用 |
| P2 | 存储路径可配置 | 高级用户需求 |
