# Bookmark API 产品需求文档

## 1. 项目概述

### 1.1 背景
开发者和信息工作者在日常工作中会收藏大量网页链接，但浏览器书签管理功能有限，难以通过 API 进行自动化管理。本项目提供一个轻量级的个人书签收藏 REST API，作为浏览器扩展、命令行工具或其他客户端的后端服务，让用户通过 HTTP 接口统一管理收藏的网页链接和标签分类。

### 1.2 目标用户
有自建工具习惯的个人用户（开发者、信息工作者），单用户场景，无需多用户认证。

### 1.3 核心价值
通过 REST API 管理个人收藏的网页链接和标签分类，为各类客户端提供统一的书签数据后端。

## 2. 功能需求

### 2.1 书签管理

**描述**: 提供书签的增删改查功能，支持分页浏览、按标签筛选和关键词搜索。

**功能点**:
- [ ] 添加书签：提交 URL、标题、描述来创建一条书签记录
- [ ] 查询书签列表：分页查看所有书签，支持按标签筛选和关键词搜索
- [ ] 查看书签详情：获取单条书签的完整信息（含关联的标签列表）
- [ ] 更新书签：修改书签的标题、URL 或描述
- [ ] 删除书签：删除书签及其与标签的关联关系

**业务规则**:
- URL 必须是合法的 HTTP/HTTPS 格式
- URL 不允许重复（同一个网页只收藏一次）
- 标题最长 200 个字符，描述最长 1000 个字符
- 删除书签时自动清除该书签与所有标签的关联记录
- 书签列表默认按创建时间倒序排列（最新的在前）

**验收标准**:
- [ ] 用户能通过 POST /bookmarks 添加一条书签，返回完整的书签信息
- [ ] 用户能通过 GET /bookmarks 获取分页书签列表，默认第 1 页每页 20 条
- [ ] 用户能通过 GET /bookmarks?tag=前端 筛选出关联了"前端"标签的书签
- [ ] 用户能通过 GET /bookmarks?keyword=TypeScript 搜索标题或描述中包含关键词的书签
- [ ] 用户能通过 GET /bookmarks/:id 获取书签详情，响应中包含关联的标签列表
- [ ] 用户能通过 PUT /bookmarks/:id 更新书签的标题、URL 或描述
- [ ] 用户能通过 DELETE /bookmarks/:id 删除书签，同时该书签与标签的关联被清除
- [ ] 添加重复 URL 时返回 409 冲突错误
- [ ] 操作不存在的书签 ID 时返回 404 错误

### 2.2 标签管理

**描述**: 提供标签的增删查功能，以及书签与标签的关联管理。

**功能点**:
- [ ] 创建标签：创建一个新的标签
- [ ] 查询所有标签：获取标签列表，每个标签附带关联的书签数量
- [ ] 删除标签：删除标签并自动解除其与所有书签的关联
- [ ] 给书签添加标签：为指定书签关联一个标签
- [ ] 移除书签的标签：解除指定书签与指定标签的关联

**业务规则**:
- 标签名不允许重复（大小写不敏感，统一存储为小写）
- 标签名最长 50 个字符，只允许中文、英文字母、数字、连字符
- 删除标签时自动清除该标签与所有书签的关联记录
- 同一个书签不能重复关联同一个标签
- 一个书签最多关联 10 个标签

**验收标准**:
- [ ] 用户能通过 POST /tags 创建标签
- [ ] 用户能通过 GET /tags 获取所有标签及每个标签关联的书签数量
- [ ] 用户能通过 DELETE /tags/:id 删除标签，同时解除与所有书签的关联
- [ ] 用户能通过 POST /bookmarks/:id/tags 为书签添加标签
- [ ] 用户能通过 DELETE /bookmarks/:id/tags/:tagId 移除书签上的标签
- [ ] 创建重复标签名时返回 409 冲突错误
- [ ] 同一书签重复关联同一标签时返回 409 冲突错误
- [ ] 书签已有 10 个标签时再添加标签返回 400 错误

## 3. 非功能需求

- 性能: 单次 API 响应时间 < 100ms（1000 条书签、100 个标签规模下）
- 存储: 使用 SQLite 本地存储，数据文件保存在项目目录下
- 兼容性: 支持 macOS 和 Linux

## 4. 不在范围内

- 用户认证和权限管理
- 多用户支持
- 云同步和数据导入/导出
- 前端页面或管理界面
- 书签缩略图 / 网页快照
- 书签访问统计

## 5. 优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 书签 CRUD（添加、查询列表、查看详情、更新、删除） | 最基本的书签管理能力，缺一不可 |
| P0 | 标签 CRUD（创建、查询列表、删除） | 标签是核心分类手段 |
| P0 | 书签-标签关联管理（添加标签、移除标签） | 关联是标签功能的核心价值 |
| P1 | 按标签筛选书签 | 标签的主要使用场景 |
| P1 | 关键词搜索书签 | 书签多了之后的核心需求 |
| P2 | 批量删除书签 | 便利性功能，非必须 |
| P2 | 批量为书签添加/移除标签 | 便利性功能，非必须 |
