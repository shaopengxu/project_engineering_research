# {项目名称} 接口契约

> 根据项目类型选择对应的模板格式。无论哪种类型，每个接口都必须包含四要素：**输入、输出、业务规则、错误处理**。

## ⚠️ 使用说明

**本文件包含 5 种项目类型的模板，使用时请：**

1. **选择一个**匹配你项目类型的模板（A-E）
2. **删除其余 4 个模板**，只保留选中的
3. 删除本"使用说明"段落
4. 将选中模板的标题改为你的项目名称

| 模板 | 适用项目类型 | 典型场景 |
|------|------------|---------|
| **A: HTTP REST API** | Web 后端、全栈应用、BFF 层 | Express / FastAPI / Spring Boot |
| **B: CLI 工具** | 命令行工具、脚手架、DevOps 工具 | Commander.js / Click / Cobra |
| **C: SDK / 库** | 工具库、SDK、共享 package | npm 包 / PyPI 包 / Go module |
| **D: 消息队列 / 事件驱动** | 消息消费者、事件处理器 | Kafka / RabbitMQ / SQS |
| **E: 纯前端应用** | SPA、管理后台 | React / Vue / Next.js（无自建后端） |

如果项目混合了多种类型（如后端 API + CLI 管理工具），可以保留多个模板，按模块分别使用。
